# 芯片验证 Tracker 用户手册

> **当前版本**: v0.10.1 | **更新日期**: 2026-03-23

---

## 1. 系统简介

芯片验证 Tracker 是一款专为芯片验证团队设计的项目管理工具，用于跟踪和管理验证过程中的 Cover Points (CP) 和 Test Cases (TC)。

**主要功能：**
- 项目管理：创建、切换、备份和恢复项目
- Cover Point 管理：添加、编辑、删除和追踪覆盖点
- **TC/CP 关联管理：建立测试用例与覆盖点的多对多关联，未关联项高亮显示**
- Test Case 管理：添加、编辑、删除和追踪测试用例
- **状态日期跟踪：自动记录 CODED/FAIL/PASS/REMOVED 状态变更日期**
- **进度图表：可视化展示计划曲线与实际覆盖率**
- **快照管理：定期采集项目进度快照，支持导出**
- 批量操作：批量更新状态、目标日期、DV Milestone
- 数据导入/导出：支持 Excel 和 CSV 格式
- **用户反馈：提交问题和建议给开发团队**
- **批量创建用户：管理员可批量创建普通用户账号**

## 2. 登录说明

### 登录方式
1. **账号登录**：使用用户名和密码登录
2. **访客登录**：点击"访客登录"按钮快速体验

### 角色权限
| 角色 | 权限 |
|------|------|
| admin | 全部权限，包括用户管理、快照管理、删除项目 |
| user | 正常操作，包括导入/导出、创建/编辑/删除 CP/TC |
| guest | 只读访问，无法修改数据 |

### 首次使用
1. 使用管理员账号登录
2. 在"用户管理"中添加新用户
3. 创建或选择项目开始使用

## 3. 项目管理

### 创建项目
1. 点击顶部工具栏的「项目」按钮
2. 在弹出的对话框中输入项目名称
3. 设置项目起止日期（必填）
4. 点击「创建」按钮

### 切换项目
- 点击顶部项目下拉菜单选择已有项目
- 项目数据相互独立

### 编辑项目
- 在项目列表中点击「编辑」按钮
- 可修改项目名称和起止日期

### 备份与恢复
1. 点击顶部工具栏的「备份」按钮
2. **导出备份**：点击「导出备份」按钮下载项目数据
3. **从备份恢复**：上传备份文件或从默认目录选择

### 删除项目
- 在项目列表中点击「删除」按钮
- 删除前系统会自动创建归档备份
- 仅 admin 角色可删除项目

## 4. Cover Point 管理

### 添加 Cover Point
1. 在 Cover Points 标签页
2. 点击「+ 添加 CP」按钮
3. 填写以下信息：
   - **Feature** (必填)：功能模块
   - **Sub-Feature**：子功能模块
   - **Cover Point** (必填)：覆盖点名称
   - **Priority**：优先级 (P0/P1/P2)
   - **Cover Point Details** (必填)：详细描述
   - **Comments**：备注

### 查看 CP 详情
- 点击「详情」按钮展开查看详细信息和关联的 Test Cases
- 点击「详情 ▼」可展开/收起所有 CP 的详情

### 编辑/删除 CP
- 点击「编辑」修改 CP 信息
- 点击「删除」移除 CP（会同时清除与 TC 的关联）

### 过滤与搜索
- **文本搜索**：输入关键字搜索
- **Feature 过滤**：按功能模块筛选
- **Priority 过滤**：按优先级筛选
- **关联状态过滤**：可筛选「未关联」的 CP
- 点击「重置」清除所有过滤条件

### 覆盖率显示
每个 CP 显示关联 TC 的 PASS 覆盖率：
| 覆盖率 | 显示 |
|--------|------|
| 100% | 🟢 100% (3/3) |
| 部分 | 🟡 66.7% (2/3) |
| 0% | ⚪ 0% (0/3) |
| 无关联 | ⚪ 0% (0/0) |

### 未关联高亮显示
**未关联的 CP 显示为红色加粗 + 🔗图标**，提醒用户及时关联测试用例。

## 5. Test Case 管理

### 添加 Test Case
1. 切换到 Test Cases 标签页
2. 点击「+ 添加 TC」按钮
3. 填写以下信息：
   - **DV Milestone**：验证里程碑 (DV0.3/DV0.5/DV0.7/DV1.0)
   - **Target Date**：目标完成日期
   - **TestBench** (必填)：测试台名称
   - **Category**：测试类别
   - **Owner**：负责人
   - **Test Name** (必填)：测试名称
   - **Scenario Details** (必填)：场景描述
   - **Checker Details**：检查器详情（可隐藏）
   - **Coverage Details**：覆盖详情（可隐藏）
   - **Comments**：备注（可隐藏）
   - **关联 Cover Points**：选择关联的 CP

### 状态说明
| 状态 | 说明 | Status Date 记录 |
|------|------|------------------|
| OPEN | 待开始 | - |
| CODED | 已编码 | 自动记录 CODED 日期 |
| FAIL | 失败 | 自动记录 FAIL 日期 |
| PASS | 通过 | 自动记录 PASS 日期 |
| REMOVED | 已移除 | 自动记录 REMOVED 日期，解除 CP 关联 |

### 状态日期自动记录
变更 TC 状态时，系统会自动记录变更日期：
- OPEN → CODED：记录 coded_date
- 任何 → FAIL：记录 fail_date
- 任何 → PASS：记录 pass_date
- 任何 → REMOVED：记录 removed_date，并自动清除与 CP 的关联

### 编辑/删除 TC
- 点击「编辑」修改 TC 信息
- 点击「删除」移除 TC

### 未关联高亮显示
**未关联的 TC 显示为红色加粗 + 🔗图标**，提醒用户及时关联覆盖点。

### 批量操作
- **批量更新状态**：选中多个 TC，一键更新状态
- **批量修改 Target Date**：批量设置目标日期
- **批量修改 DV Milestone**：批量设置验证里程碑

### 过滤与排序
- **Status 过滤**：按状态筛选
- **DV Milestone 过滤**：按验证里程碑筛选
- **Owner 过滤**：按负责人筛选
- **Category 过滤**：按类别筛选
- **排序**：按 ID、TestBench、Owner、状态排序

## 6. TC/CP 关联管理

### 关联概念
- 一个 TC 可以关联多个 CP
- 一个 CP 可以被多个 TC 关联
- 这是多对多的关系

### 关联 TC 到 CP
**方式一：编辑 TC 时关联**
1. 点击 TC 行的「编辑」按钮
2. 在「关联 Cover Points」下拉框中选择要关联的 CP
3. 支持多选（按住 Ctrl/Cmd 可选择多个）
4. 点击「保存」

**方式二：在 CP 详情中查看关联 TC**
1. 点击 CP 行的「详情」按钮
2. 展开后显示该 CP 关联的所有 TC
3. 显示 TC 的 ID、名称和状态

### 关联状态可视化
- **已关联**：正常显示
- **未关联**：显示为红色加粗 + 🔗图标

### 查看关联覆盖率
在 CP 列表中，每个 CP 显示关联 TC 的 PASS 覆盖率：
- 🟢 100%：全部关联 TC 状态为 PASS
- 🟡 xx%：部分关联 TC 状态为 PASS
- ⚪ 0%：无关联 TC 或全部未 PASS

## 7. 进度图表 (Progress Charts)

Tracker 提供可视化进度跟踪功能，帮助团队监控验证进度。

### 访问进度图表
1. 选择一个项目
2. 点击顶部 Tab 栏的「Progress Charts」标签

### 图表内容
进度图表展示两种曲线：

**计划曲线（蓝色虚线）**
- 基于 TC 的 Target Date 计算每周预期覆盖率
- 反映项目规划进度

**实际曲线（绿色实线）**
- 基于当前 PASS 状态 TC 关联的 CP 覆盖率
- 反映项目实际进度

### 时间段选择
- 可设置起始日期和结束日期
- 点击「应用」按钮刷新图表
- 点击「清空」恢复显示全周期数据

### Priority 过滤 (v0.10.0+)
- 支持按 CP Priority (P0/P1/P2/P3) 过滤图表数据
- 多值过滤时显示加权平均覆盖率

## 8. 快照管理

快照功能用于定期记录项目进度状态。

### 手动创建快照
1. 进入 Progress Charts 页面
2. 点击「刷新快照」按钮
3. 系统自动采集当前项目状态

### 快照数据内容
- 快照日期
- 实际覆盖率
- TC 统计数据（总数、PASS 数量）
- CP 统计数据（总数、已覆盖数量）
- **各 Priority 覆盖率** (P0/P1/P2/P3)

### 查看快照列表
1. 点击「快照管理」按钮
2. 弹出窗口显示所有历史快照
3. 列表按日期倒序排列

### 删除快照
- 在快照列表中点击「删除」按钮
- 仅 admin 角色可删除快照

### 导出快照数据
1. 在快照管理对话框中点击「导出进度数据」按钮
2. 支持 CSV 和 JSON 格式
3. 下载文件包含所有快照记录

## 9. 导入/导出功能

### 导入数据
1. 点击「📥 导入 CP」或「📥 导入 TC」按钮
2. 选择文件（支持 .xlsx 和 .csv 格式）
3. 可点击「下载导入模板」获取模板文件
4. 预览数据后点击「导入」

### 导出数据
1. 点击「📤 导出 CP」或「📤 导出 TC」按钮
2. 选择导出格式（Excel 或 CSV）
3. 点击「导出」下载文件

### 数据格式
**CP 导入字段：**
- Feature, Sub-Feature, Cover Point, Priority, Cover Point Details, Comments

**TC 导入字段：**
- DV Milestone, TestBench, Category, Owner, Test Name, Scenario Details, Checker Details, Coverage Details, Comments, Target Date, Status

## 10. 用户管理 (admin)

### 创建用户
1. 点击顶部「用户管理」按钮
2. 点击「添加用户」按钮
3. 填写用户名、密码，选择角色
4. 点击「创建」

### 批量创建用户 (v0.10.1)
1. 点击顶部「用户管理」按钮
2. 点击「批量创建」按钮
3. 输入用户名列表（每行一个）
4. 点击「创建」
5. 系统统一设置密码为 `123456`

### 编辑/禁用/删除用户
- 点击用户行的操作按钮进行管理
- 禁用的用户无法登录

### 重置用户密码
1. 在用户列表中点击「重置密码」按钮
2. 系统生成随机密码并显示

2. 在弹出的对话框中切换到「反馈」标签页

### 反馈类型
| 类型 | 说明 |
|------|------|

## 12. 常见问题

### Q: 如何提高 Cover Point 覆盖率？
A: 为每个 CP 编写足够的 Test Cases，使关联的 TC 状态为 PASS，即视为覆盖。

### Q: 为什么无法导入数据？
A: 请检查：
1. 文件格式是否为 .xlsx 或 .csv
2. 必填字段是否完整
3. 字段值是否合法（如 Priority 只能是 P0/P1/P2）

### Q: 如何给用户分配权限？
A: 使用 admin 账号登录，在「用户管理」中添加新用户并设置角色。

### Q: 数据误删如何恢复？
A: 如果有备份，可以在「备份」功能中恢复数据。

### Q: 页面显示异常怎么办？
A: 尝试：
1. 刷新浏览器
2. 清除浏览器缓存
3. 使用 Chrome 或 Edge 浏览器

### Q: 什么是 TC/CP 关联？
A: 一个测试用例 (TC) 可以关联多个覆盖点 (CP)，一个 CP 也可以被多个 TC 关联。未关联的项会显示红色加粗 + 🔗图标提醒。

### Q: 状态日期有什么作用？
A: 每次变更 TC 状态（CODED/FAIL/PASS/REMOVED）时，系统会自动记录变更日期，方便跟踪测试进度。

### Q: 进度图表中的计划曲线和实际曲线有什么区别？
A: 计划曲线基于 TC 的 Target Date 计算预期覆盖率，实际曲线基于当前 PASS 状态 TC 关联的 CP 计算实际覆盖率。
