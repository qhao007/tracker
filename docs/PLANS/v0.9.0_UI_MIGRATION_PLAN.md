# v0.9.0 UI 样式迁移计划

## 1. 需求分析

### 背景
- v0.9.0 已完成设计系统 (`design-system.css`，1072 行 CSS 变量和组件样式)
- 现有 `index.html` 中仍有 174 行 inline CSS 样式未迁移
- 需要将旧样式逐步迁移到使用 CSS 变量，实现视觉统一

### 目标
- 将 `index.html` 中的 inline CSS 逐步迁移到使用 CSS 变量
- 确保每次提交可独立上线，不影响现有功能
- 保持向后兼容，逐步替换而非一次性重构

### 范围
- **包含**：index.html 中的 `<style>` 块样式迁移
- **不包含**：JavaScript 代码修改、API 变更、功能重构

## 3. 实施步骤

### T1: Header 样式迁移
**目标**: 将 Header 相关样式改为使用 CSS 变量

**需要修改的类**:
- `.header` → 使用 `var(--color-primary)` 渐变
- `.header-btn` → 使用 CSS 变量
- `.header-project-selector` → 使用 CSS 变量

**步骤**:
1. 在 `design-system.css` 中添加 Header 样式（如果尚未添加）
2. 在 `index.html` `<style>` 中标记为"迁移完成"（注释）
3. 移除旧样式，让其使用 `design-system.css` 中的定义

**验证**: 刷新页面，Header 样式保持一致

---

### T2-T3: Tabs 和 Toolbar 样式迁移
**目标**: 使用 CSS 变量统一样式

## 5. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 样式冲突 | 页面布局错乱 | 每次只迁移一类，验证后再继续 |
| 遗漏样式 | 功能异常 | 保留旧样式作为兜底 |
| 回归问题 | 原有功能损坏 | 每次提交后测试版验证 |
| 时间不足 | 计划延期 | 优先完成 P0-P1 任务 |

---

## 6. 实施原则

1. **小步快跑**: 每次只迁移一个组件类别的样式
2. **验证优先**: 每次提交后验证测试版正常运行
3. **向后兼容**: 保留旧样式作为兜底，逐步移除
4. **增量迁移**: 不要求一次完成所有迁移，可分多次迭代

---

## 7. 验收标准

- [ ] 所有页面组件使用 CSS 变量
- [ ] 每次提交后测试版正常运行
- [ ] 样式显示与迁移前一致
- [ ] 无新增 CSS 警告
