## 3. 需求详细分析

### REQ-001: CP 页面覆盖率左右布局显示

#### 3.1.1 需求描述

当前 CP 表格中，每一行的覆盖率信息（实际覆盖率 + 关联 TC 数量）是上下分两行显示，导致每行 CP 高度较大。期望改为左右显示，减少行高。

#### 3.1.2 当前实现

**文件**: `dev/index.html` 第 1354 行

```javascript
html += `<td style="padding: 8px; border: 1px solid #ddd;">
  <span class="coverage-badge ${...}">${cp.coverage}%</span>
  <small class="coverage-detail">${cp.coverage_detail}</small>
</td>`;
```

**显示效果**: 上下排列
```
  ┌─────────────┐
  │    85%     │  ← coverage-badge
  │ (3 TC)     │  ← coverage-detail (小字)
  └─────────────┘
```

#### 3.1.3 期望实现

改为左右排列：
```
  ┌─────────────┐
  │ 85%  (3 TC)│  ← 同一行
  └─────────────┘
```

#### 3.1.4 方案选项

| 方案 | 优点 | 缺点 | 预估工时 |
|------|------|------|----------|
| **方案A**: CSS 修改 + 内联显示 | 改动最小 | 需调整 CSS 样式 | 0.5h |
| 方案B: 合并为一个 Badge | 显示统一 | 可能需要改数据结构 | 1h |
| 方案C: 分列显示 | 布局清晰 | 占用更多列宽 | 1.5h |

**推荐方案**: 方案A - 使用 CSS Flexbox 左右布局

#### 3.1.5 验收标准

- [ ] CP 表格每行覆盖率信息为左右显示
- [ ] 不影响其他表格列的布局
- [ ] 移动端显示正常

### REQ-002: 关于对话框增加用户反馈功能

#### 3.2.1 需求描述

在"关于"对话框中增加用户反馈功能，允许用户：
1. 填写反馈条目（问题描述、类型、优先级）
2. 增加新的反馈条目
3. 提交反馈后自动生成需求文档
4. 文档保存到 `feedbacks/new/` 目录

#### 3.2.2 需求分析

**功能拆解**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| F1 | 关于对话框增加"反馈"标签页 | P1 |
| F2 | 反馈表单：类型选择（Bug/功能/优化） | P1 |
| F3 | 反馈表单：描述文本框 | P1 |
| F4 | 反馈表单：新增条目按钮 | P2 |
| F5 | 反馈列表：显示已填写的反馈 | P2 |
| F6 | 提交功能：生成 JSON 需求单 | P1 |
| F7 | 自动保存到 feedbacks/new/ | P1 |

#### 3.2.3 方案选项

| 方案 | 描述 | 优点 | 缺点 | 预估工时 |
|------|------|------|------|----------|
| **方案A** | 前端表单 + API 保存 | 交互流畅 | 需要新增 API | 5h |
| 方案B | 前端生成 JSON + 邮件发送 | 无需后端 | 需要配置邮件 | 4h |
| 方案C | 仅前端 JSON 下载 | 最简实现 | 需用户手动上传 | 2h |

**推荐方案**: 方案A - 前端表单 + API 保存

**原因**: 
- 用户体验最好
- 与现有 feedback 目录结构一致
- 便于后续处理

#### 3.2.4 技术方案

**前端**:
1. 在 versionModal 中添加"反馈"标签页
2. 反馈表单组件：类型下拉 + 描述文本框 + 提交按钮
3. 反馈列表组件：显示已填写的反馈（支持多条）

**后端**:
1. 新增 API: `POST /api/feedback`
2. 保存位置: `/projects/management/feedbacks/new/FEEDBACK_{timestamp}.json`

**数据格式**:
```json
{
  "id": "FEEDBACK_001",
  "type": "bug|feature|optimization",
  "title": "简短描述",
  "description": "详细描述",
  "priority": "P1|P2|P3",
  "created_at": "2026-03-07T12:00:00Z",
  "user": "admin"
}
```

#### 3.2.5 验收标准

- [ ] 关于对话框有"反馈"标签页
- [ ] 可以填写反馈表单
- [ ] 可以提交反馈
- [ ] 提交后在 feedbacks/new/ 目录生成 JSON 文件
- [ ] 提交成功有提示
