## 2. 需求 1: 用户登录系统

### 2.1 功能概述

**功能名称**: 用户认证系统

**背景/原因**:
- 网页访问需要支持用户登录
- 只有通过认证后才可访问网页内容
- 区分管理员、普通用户、访客三种角色

### 2.2 角色定义

| 角色 | 登录方式 | 说明 |
|------|----------|------|
| **admin** | 用户名+密码 | 管理员，完全控制 |
| **user** | 用户名+密码 | 普通用户，可创建/编辑/删除 TC/CP |
| **guest** | 访客登录按钮（无需密码） | 访客，仅查看，不可手动创建 |

### 2.3 数据库设计

**存储位置**: `data/users.db`（独立数据库，与项目数据分离）

**用户表 (users)**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名（唯一） |
| password_hash | TEXT | 密码哈希（guest 可为 NULL） |
| role | TEXT | 角色：admin/user/guest |
| is_active | INTEGER | 是否启用（1=启用，0=禁用） |
| must_change_password | INTEGER | 首次登录是否必须修改密码 |
| created_at | DATETIME | 创建时间 |
| last_login | DATETIME | 最后登录时间 |

### 2.4 暴力破解防护

**存储位置**: `data/login_attempts.json`

**规则**:
- 密码连续错误 5 次后，锁定 15 分钟
- 登录成功或 15 分钟后自动清除记录

### 2.5 登录界面

```
+--------------------------------------------------+
|              芯片验证 Tracker                    |
|                                                  |
|           用户名: [____________]                 |
|           密码:   [____________]                 |
|                                                  |
|              [ 登录 ]                           |
|                                                  |
|         [ 访客登录 ]                            |
+--------------------------------------------------+
```

### 2.6 Session 配置

```python
app.config['SESSION_COOKIE_HTTPONLY'] = True    # 防止 XSS 读取
app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'   # 防止 CSRF
app.config['PERMANENT_SESSION_LIFETIME'] = 86400 # 24小时
```

|------|--------|------|------|
| 管理员 | admin | admin123 | admin |
| 访客 | guest | (无密码) | guest |

## 4. 需求 3: 项目删除功能

### 4.1 功能概述

**功能名称**: 项目删除

**目标用户**: 管理员用户

### 4.2 删除流程

```
1. 自动创建归档备份（JSON 格式）
2. 标记项目为已归档（is_archived = True）
3. 删除项目数据库文件
4. 返回成功
```

### 4.3 删除确认对话框

```
+--------------------------------------------------+
|  确认删除                                        |
|  ------------------------------------------------|
|  确定要删除项目 "Project A" 吗？                |
|  此操作将同时删除项目下的所有 Cover Points       |
|  和 Test Cases。此操作不可恢复。                |
|                                                  |
|           [ 取消 ]    [ 确认删除 ]              |
+--------------------------------------------------+
```

## 6. API 接口

### 6.1 认证 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/auth/login` | 用户名密码登录 | 公开 |
| POST | `/api/auth/guest-login` | 访客登录 | 公开 |
| POST | `/api/auth/logout` | 登出 | 登录 |
| GET | `/api/auth/me` | 获取当前用户信息 | 登录 |

### 6.2 用户管理 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/users` | 获取用户列表 | 管理员 |
| POST | `/api/users` | 创建用户 | 管理员 |
| PATCH | `/api/users/{id}` | 更新用户（禁用/启用） | 管理员 |
| DELETE | `/api/users/{id}` | 删除用户 | 管理员 |
| POST | `/api/users/{id}/reset-password` | 重置密码 | 管理员 |

### 6.3 项目 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/projects` | 获取项目列表 | 登录 |
| POST | `/api/projects` | 创建项目 | 管理员 |
| DELETE | `/api/projects/{id}` | 删除项目 | 管理员 |

### 6.4 Cover Point API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/cp` | 获取 CP 列表 | 登录 |
| POST | `/api/cp` | 创建 CP | 登录（非 guest） |
| PUT | `/api/cp/{id}` | 更新 CP | 登录（非 guest） |
| DELETE | `/api/cp/{id}` | 删除 CP | 登录（非 guest） |

### 6.5 Test Case API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/tc` | 获取 TC 列表 | 登录 |
| POST | `/api/tc` | 创建 TC | 登录（非 guest） |
| PUT | `/api/tc/{id}` | 更新 TC | 登录（非 guest） |
| DELETE | `/api/tc/{id}` | 删除 TC | 登录（非 guest） |

### 6.6 导入/导出 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/import/template/{type}` | 获取导入模板 | 登录 |
| POST | `/api/import/{type}` | 导入数据 | 登录（非 guest） |
| GET | `/api/export/{type}` | 导出数据 | 登录（非 guest） |
