# v0.8.1 版本实施计划

## 1. 需求分析

**背景**: v0.8.0 已完成基础图表框架，v0.8.1 需要实现 CP 覆盖率计划曲线展示功能

**目标**:
- 实现基于 TC target_date 的计划曲线计算
- 在 Progress Charts 中展示计划曲线
- 优化 tracker_ops.py 的兼容性测试脚本

**范围**:
- 包含：
  - CP 覆盖率计划曲线计算（后端算法）
  - API 端点 `/api/progress/<project_id>` 返回计划曲线数据
  - 前端图表渲染计划曲线
  - 时间段选择器（起始/结束日期过滤）
  - tracker_ops.py 兼容性测试优化
- 不包含：
  - 实际曲线（v0.8.2）
  - 快照管理（v0.8.2）

## 3. 实施步骤

### T1: 后端覆盖率计算算法

**目标**: 实现基于 TC target_date 的覆盖率计算逻辑

**步骤**:
1. 分析需求文档中的 SQL 算法
2. 在 `api.py` 中实现 `calculate_planned_coverage(project_id, start_date, end_date)` 函数
3. 实现按周分组的算法
4. 实现 Pass 状态 TC 关联 CP 的去重计算
5. 单元测试验证算法正确性

**验证**:
- 使用 EX5 项目数据测试
- 对比手动计算结果

---

### T2: 后端 API 完善

**目标**: 扩展 `/api/progress/<project_id>` 返回计划曲线数据

**步骤**:
1. 修改 `get_progress` 函数
2. 添加查询参数支持（start_date, end_date）
3. 调用 T1 实现的计算函数
4. 返回符合数据结构的 JSON

**验证**:
- API 返回数据结构正确
- 前端能正确解析

---

### T3: 前端计划曲线渲染

**目标**: 在 Progress Charts 中渲染计划曲线

**步骤**:
1. 分析现有 Chart.js 集成代码
2. 修改 `loadProgressChart()` 函数
3. 调用 API 获取计划曲线数据
4. 配置 Chart.js 渲染折线图
5. 设置曲线样式（颜色、标签）

**验证**:
- 图表正确显示
- 数据点正确

---

### T4: 前端时间段选择器

**目标**: 添加日期范围过滤功能

**步骤**:
1. 在 HTML 中添加日期选择器控件
2. 实现日期选择事件处理
3. 调用 API 时传递日期参数
4. 图表根据过滤条件重新渲染

**验证**:
- 选择日期后图表更新
- 无效日期有错误提示

---

### T5: 前端图例和 Tooltip

**目标**: 优化用户体验

**步骤**:
1. 配置 Chart.js 图例
2. 实现自定义 tooltip 格式化
3. 显示周日期和覆盖率百分比

**验证**:
- 鼠标悬停显示详细信息

---

### T6: 边界处理

**验证**:
