## 3. 测试用例详细说明

### 3.1 API-FILTER-001: 获取全部CP（默认）

**测试目标**: 不传 filter 参数时返回所有 CP

**测试步骤**:
1. 创建测试项目
2. 创建多个 CP
3. 调用 GET /api/cp?project_id={id} 不带 filter 参数
4. 验证返回所有 CP

**预期结果**: 返回所有 CP

**实际结果**: ✅ 通过

---

### 3.2 API-FILTER-002: 获取未关联CP

**测试目标**: filter=unlinked 时只返回 linked=false 的 CP

**测试步骤**:
1. 创建两个 CP（一个未关联，一个关联到 TC）
2. 调用 GET /api/cp?project_id={id}&filter=unlinked
3. 验证只返回未关联的 CP

**预期结果**: 只返回未关联的 CP

**实际结果**: ✅ 通过

---

### 3.3 API-FILTER-003: 指定项目过滤

**测试目标**: 指定 project_id 参数时按项目过滤

**测试步骤**:
1. 创建两个不同项目
2. 为每个项目创建对应的 CP
3. 分别调用 GET /api/cp?project_id={project1_id} 和 GET /api/cp?project_id={project2_id}
4. 验证每个查询只返回对应项目的 CP

**预期结果**: 只返回指定项目的 CP

**实际结果**: ✅ 通过

---

### 3.4 API-FILTER-004: 无效filter参数

**测试目标**: 传入无效的 filter 参数时返回默认行为（全部）

**测试步骤**:
1. 创建测试 CP
2. 调用 GET /api/cp?project_id={id}&filter=invalid_value
3. 验证返回默认行为（全部 CP）

**预期结果**: 返回错误或默认全部

**实际结果**: ✅ 通过（返回默认全部 CP）
