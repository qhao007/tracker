### 2.4 JavaScript 语法错误修复

**问题**：浏览器控制台报错 `Uncaught SyntaxError: Unexpected token '/' (at app_constants.js:82:10)`

**根因**：JavaScript 对象键名不能包含 `/` 字符（会被解析为除法运算符）

```javascript
// 错误写法
N/A: 'N/A'  // / 被解析为除法

// 正确写法
'N/A': 'N/A'  // 使用引号包裹
```

**修复**：`static/js/app_constants.js` L82

```javascript
// 修改前
N/A: 'N/A',

// 修改后
'N/A': 'N/A',
```

## 2.5 CSS 加载问题修复 ✅

### 问题描述

执行 T1 迁移后，发现 design-system.css 未正确加载，页面仍显示蓝色 Header。

### 根因分析

1. `design-system.css` 位于 `/dev/` 根目录
2. Flask 默认 `/static/<path>` 路由指向不存在的 `app/static` 目录
3. 导致自定义的 `/static/<path>` 路由被默认路由覆盖

### 修复方案

**方案 A**：将 design-system.css 移至 static/css/ 目录

#### 修复步骤

1. **移动文件**
   ```bash
   mkdir -p dev/static/css
   mv dev/design-system.css dev/static/css/
   ```

2. **更新 HTML 引用** (`index.html` L7)
   ```html
   <link rel="stylesheet" href="/static/css/design-system.css">
   ```

3. **修复 Flask 路由冲突** (`app/__init__.py` L11)
   ```python
   app = Flask(__name__, static_folder=None)  # 禁用默认静态路由
   ```

### 验证结果

- [x] CSS 文件可正确访问
- [x] Header 背景：蓝色 → 紫色渐变 ✅
- [x] JavaScript 无语法错误
