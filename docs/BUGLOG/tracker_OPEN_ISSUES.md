## 使用说明

| 字段 | 说明 |
|------|------|
| **ID** | 问题编号 (ISSUE-001 开始) |
| **发现日期** | 首次发现问题的日期 |
| **优先级** | P3(低)/P2(中)/P1(高) - 不紧急但影响体验 |
| **状态** | 待处理 / 已记录 / 已修复 |
| **问题描述** | 详细描述问题和复现步骤 |
| **建议方案** | 可能的修复方向 |

### ISSUE-008

| 属性 | 内容 |
|------|------|
| **发现日期** | 2026-02-25 |
| **优先级** | P3 |
| **状态** | 待处理 |
| **问题描述** | Banner 版本号显示双重 v 前缀 (vv0.7.1)，但关于对话框版本显示正确 |
| **根因** | `index.html` 第777行: `tag.textContent = \`v${data.version} ${data.version_type}\`` - 手动加了 v 前缀，但 API 返回的 `data.version` 已包含 v。前781行关于对话框已修复为 `data.version`（无 v 前缀） |
| **建议方案** | 统一两处代码为 `data.version`（不含 v 前缀），或让 API 返回不含 v 的版本号 |

### ISSUE-004

| 属性 | 内容 |
|------|------|
| **发现日期** | 2026-02-21 |
| **优先级** | P2 |
