# Tracker 公开问题清单

> 记录平时使用中发现的小问题，待版本迭代时统一 review 和修复

---

## 使用说明

| 字段 | 说明 |
|------|------|
| **ID** | 问题编号 (ISSUE-001 开始) |
| **发现日期** | 首次发现问题的日期 |
| **优先级** | P3(低)/P2(中)/P1(高) - 不紧急但影响体验 |
| **状态** | 待处理 / 已记录 / 已修复 |
| **问题描述** | 详细描述问题和复现步骤 |
| **建议方案** | 可能的修复方向 |

## 问题列表

### ISSUE-004

| 属性 | 内容 |
|------|------|
| **发现日期** | 2026-02-22 |
| **优先级** | P1 |
| **状态** | ✅ 已修复 |
| **问题描述** | 创建项目时 ID 生成逻辑错误，使用 `len(projects) + 1` 而非 `max(id) + 1`，导致多项目共享同一 ID，产生数据冲突。2026-02-21 23:03 创建的 "test" 项目错误地使用了 ID=3，与 EX5 冲突 |
| **根本原因** | `create_project()` 函数中 ID 生成逻辑缺陷 |
| **修复方案** | 改为 `max([p["id"] for p in projects], default=0) + 1` |
| **影响范围** | 生产环境 user_data/projects.json 数据损坏 |

### ISSUE-004

| 属性 | 内容 |
|------|------|
| **发现日期** | 2026-02-21 |
| **优先级** | P2 |
| **状态** | 待处理 |
| **问题描述** | 生产环境需要支持 HTTPS 访问，当前 Session Cookie 的 SECURE 选项为 False |
| **建议方案** | 1. 购买域名并配置 SSL 证书（推荐 Let's Encrypt，免费）<br>2. 使用 Nginx 反向代理配置 HTTPS<br>3. 将 SESSION_COOKIE_SECURE 设为 True |
| **详细说明** | 当前配置为宽松模式（HTTP 可用），适用于内网/开发环境。外网访问建议配置 HTTPS。<br><br>**配置步骤**：<br>1. 购买域名（腾讯云）<br>2. 安装 Nginx + Certbot<br>3. 获取 SSL 证书<br>4. 配置 Nginx 反向代理<br><br>**相关配置**：<br>- SESSION_COOKIE_SECURE = True（需 HTTPS）<br>- SESSION_COOKIE_HTTPONLY = True<br>- SESSION_COOKIE_SAMESITE = 'Lax' |
