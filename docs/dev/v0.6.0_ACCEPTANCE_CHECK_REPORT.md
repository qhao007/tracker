## 待修复问题

### BUG-016: PASS → 其他状态缺少二次确认 ⚠️ High

**验收项**:
- 5.1.4: PASS → 其他状态有二次确认，提示"完成日期会被重置"
- 5.4.4: PASS → 其他状态有二次确认

**严重性**: High

**问题描述**:
规格书要求当状态从 PASS 改为其他状态时，需要二次确认并提示"完成日期会被重置"。

**当前状态**:
`updateTCStatus()` 函数直接调用 API，没有确认逻辑。

**修复优先级**: P0

---

### BUG-017: DV Milestone 默认值不正确 ⚠️ Medium

**验收项**: 5.5.3 默认值 DV1.0

**严重性**: Medium

**问题描述**:
前端 Modal 中 DV Milestone 下拉框的默认选项是 "-- 选择 --" (空值)，而不是 DV1.0。

**修复优先级**: P1
