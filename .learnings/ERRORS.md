## 2026-02-22

### 错误：违反安全铁律 - 开发环境触碰生产数据

| 属性 | 值 |
|------|-----|
| **日期** | 2026-02-21 23:03 |
| **严重性** | Critical |
| **类型** | 安全违规 |

**错误描述**: 
在 v0.7.1 开发过程中，通过 8081 测试端口（运行 dev 代码）创建了名为 "test" 的项目，错误地写入了生产环境的 `user_data` 目录，导致：
1. 新项目 ID=3 与 EX5 (ID=3) 冲突
2. 生产数据被意外修改

**根本原因**:
1. **未区分端口用途**：
   - 8080 = 生产环境 → user_data
   - 8081/8082 = 测试/开发 → 应使用 test_data
   
2. **运行时目录问题**：
   - gunicorn 8081 实际运行在 `/projects/management/tracker/dev/`
   - dev/data 是软链接 → test_data ✅
   - 但 8080 的代码在 `/release/tracker/v0.7.0/`
   - release/v0.7.0/data 是软链接 → user_data ❌

3. **缺少环境隔离检查**：
   - 开发时没有确认当前操作的数据目录

**影响**:
- 生产环境 `projects.json` 包含重复 ID
- 数据完整性受损
- 需要手动修复

**正确的做法**:
- 开发/测试只能使用 8081/8082 端口
- 8080 是生产禁区，任何代码修改都不能在上面测试
- 如需验证生产行为，应该在测试环境完全通过后再发布
